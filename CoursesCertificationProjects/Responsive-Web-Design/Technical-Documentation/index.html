Introduction

React is a JavaScript library for building user interfaces, particularly web applications. It was developed by Facebook and is now maintained by Meta and the open-source community.

React allows developers to create large web applications that can change data, without reloading the page. The main purpose of React is to be fast, scalable, and simple.

React uses a declarative programming paradigm, which means you describe what the UI should look like for any given state, and React will efficiently update and render just the right components when your data changes.

The library is component-based, allowing you to build encapsulated components that manage their own state, then compose them to make complex UIs.

import React from 'react';
  • Declarative: React makes it painless to create interactive UIs
  • Component-Based: Build encapsulated components that manage their own state
  • Learn Once, Write Anywhere: Develop new features without rewriting existing code
Getting Started

To get started with React, you need to have Node.js installed on your computer. Once you have Node.js, you can use Create React App to set up a new React project quickly.

Create React App is an officially supported way to create single-page React applications. It offers a modern build setup with no configuration.

npx create-react-app my-app
cd my-app
npm start

This will create a new React application in the my-app directory and start the development server.

The development server will automatically reload the page when you make changes to the source code, making development faster and more efficient.

  • Install Node.js from the official website
  • Use npm or yarn as your package manager
  • Create React App provides a good starting point for beginners
Components

Components are the building blocks of React applications. They let you split the UI into independent, reusable pieces, and think about each piece in isolation.

Components can be defined as JavaScript functions or classes. Function components are simpler and are the recommended way to write components in modern React.

function Welcome(props) {
  return <h1>Hello, {props.name}!</h1>;
}

// Or as an arrow function
const Welcome = (props) => {
  return <h1>Hello, {props.name}!</h1>;
};

Components can accept arbitrary inputs called "props" and return React elements describing what should appear on the screen.

Always start component names with a capital letter. React treats components starting with lowercase letters as DOM tags.

  • Function components are simpler and more commonly used
  • Class components have additional features like lifecycle methods
  • Components should be pure functions when possible
State and Props

Props (short for properties) are a way of passing data from parent to child components. They are read-only and should not be modified by the child component.

State is a way for components to manage their own data. Unlike props, state can be changed using the setState method in class components or the useState hook in function components. 

// Props example
function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

// State example with useState hook
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}

State should be treated as immutable. Always use the setter function to update state, never modify state directly.

  • Props flow down from parent to child components
  • State is local to the component that owns it
  • State updates trigger re-renders of the component
Hooks

Hooks are a new addition in React 16.8 that let you use state and other React features without writing a class component.

The most commonly used hooks are useState for managing state and useEffect for side effects like data fetching, subscriptions, or manually changing the DOM.

import React, { useState, useEffect } from 'react';

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  
  useEffect(() => {
    fetchUser(userId).then(setUser);
  }, [userId]);
  
  if (!user) return <div>Loading...</div>;
  
  return <div>Welcome, {user.name}!</div>;
}

Hooks follow two important rules: only call hooks at the top level of your React function, and only call hooks from React functions.

  • useState: Adds state to function components
  • useEffect: Performs side effects in function components
  • useContext: Consumes context values
  • useReducer: Alternative to useState for complex state logic
Event Handling

React events are named using camelCase, not lowercase. With JSX you pass a function as the event handler, rather than a string.

React uses SyntheticEvent, a cross-browser wrapper around the browser's native event. It has the same interface as native events, but works consistently across all browsers.

function Button() {
  function handleClick(e) {
    e.preventDefault();
    console.log('The link was clicked.');
  }

  return (
    <button onClick={handleClick}>
      Click me
    </button>
  );
}

You cannot return false to prevent default behavior in React. You must call preventDefault explicitly.

Event handlers receive a SyntheticEvent object as their first parameter, which wraps the native event and provides a consistent API across different browsers.

  • Use camelCase for event names (onClick, onChange, onSubmit)
  • Pass functions as event handlers, not strings
  • Use preventDefault() to prevent default browser behavior
Lifecycle Methods

Lifecycle methods are special methods in class components that run at specific points in a component's life. They allow you to hook into different phases of a component's existence.

The three main phases are mounting (when component is created), updating (when props or state change), and unmounting (when component is removed).

In modern React with hooks, lifecycle methods are replaced by the useEffect hook, which can handle all lifecycle scenarios.

// Class component lifecycle
class MyComponent extends React.Component {
  componentDidMount() {
    // Runs after component mounts
  }
  
  componentDidUpdate(prevProps, prevState) {
    // Runs after component updates
  }
  
  componentWillUnmount() {
    // Runs before component unmounts
  }
}

// Hook equivalent
function MyComponent() {
  useEffect(() => {
    // componentDidMount logic
    
    return () => {
      // componentWillUnmount logic
    };
  }, []); // Empty dependency array = componentDidMount
}

Common lifecycle methods include componentDidMount, componentDidUpdate, and componentWillUnmount.

  • Mounting: constructor, componentDidMount
  • Updating: componentDidUpdate, getSnapshotBeforeUpdate
  • Unmounting: componentWillUnmount
  • Error Handling: componentDidCatch, getDerivedStateFromError
